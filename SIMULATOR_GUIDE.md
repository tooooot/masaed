# محاكاة الأطراف — دليل شامل 🤖

**الميزة الجديدة**: محاكي ذكي + مساعد انتقاد وتطوير

---

## 🎯 **الفكرة الأساسية**

**قبل**: تختبر التفاوض مع أرقام حقيقية، تنتظر الرسائل يدويّاً

**الآن**: 
- 🤖 **Seeker Simulator** — محاكي AI يتقمص دور الباحث
- 🤖 **Owner Simulator** — محاكي AI يتقمص دور صاحب العرض
- 🧠 **Critic Assistant** — يقيّم المحادثة + الكود + التعليمات

---

## 🔬 **المكونات الثلاثة**

### **1️⃣ Seeker Simulator (محاكي الباحث)**

```python
# يتقمص دور الباحث الحقيقي
- يستخدم بيانات الطلب (المدينة، الميزانية، الغرف، إلخ)
- يتحدث بالعربية السعودية الطبيعية
- يكون واقعياً في التفاوض
- يسأل عن التفاصيل المهمة
- يبدأ متحفظاً، تدريجياً أكثر مرونة
```

**System Prompt:**
```
أنت باحث حقيقي عن سكن في السوق السعودي.

المعطيات:
- المدينة: {city}
- الحي: {district}
- الغرف: {rooms}
- الميزانية: {budget} ريال/سنة
- الأثاث: {furnished}

السلوك:
- ابدأ بتحية ودية
- أخبر عن احتياجاتك بشكل طبيعي
- كن واقعياً في عروضك
- اسأل عن التفاصيل المهمة
- استخدم العربية السعودية الطبيعية
```

### **2️⃣ Owner Simulator (محاكي المالك)**

```python
# يتقمص دور صاحب العرض
- يستخدم بيانات العرض (السعر، المواصفات، الشروط)
- يروّج للعقار بحماس
- يطلب سعراً عادلاً
- يسأل عن خلفية المستأجر
- يكون منفتحاً للتفاوض
```

**System Prompt:**
```
أنت مالك عقار تريد تأجيره بسعر جيد.

المعطيات:
- العقار: {title}
- السعر: {price} ريال/سنة
- المواصفات: {specs}
- الشروط: {terms}

السلوك:
- ارحب بالمستأجر بحماس
- اشرح مميزات العقار
- اطلب سعراً عادلاً
- اسأل عن خلفية المستأجر
- كن حازماً لكن منفتحاً للتفاوض
```

### **3️⃣ Critic Assistant (مساعد الانتقاد)**

يقيّم المحادثة على 4 أبعاد:

#### **أ) جودة المحادثة (1-10)**
```
- هل كانت طبيعية وواقعية؟
- هل التزم كل طرف بشخصيته؟
- هل كان التفاوض منطقياً؟
- هل وصلا لاتفاق معقول؟
```

#### **ب) أداء الوسيط (1-10)**
```
- هل فهم النوايا صحيح؟
- هل التكتيكات مناسبة؟
- هل اقترح أسعاراً معقولة؟
- هل حافظ على الحوار؟
```

#### **ج) أخطاء الكود**
```
🔴 حرجة:  خطأ يكسر النظام
🟠 متوسطة: مشكلة في المنطق
🟡 بسيطة:  تحسين صغير
```

#### **د) مشاكل التعليمات (Prompts)**
```
📝 النقص:      معلومات ناقصة
⚠️ التناقضات: تعارضات في التعليمات
💡 الاقتراحات: طرق أفضل
```

## 📊 **مثال واقعي: النتائج**

### **المحادثة المولّدة:**
```
👤 محمد: السلام عليكم ورحمة الله، أنا محمد
         أبحث عن شقة في جدة للعائلة، 
         3 غرف، ميزانيتي 2200 ريال/سنة

🏠 مالك: وعليكم السلام ورحمة الله!
         أنا عندي شقة رائعة في الجود،
         3 غرف مفروشة، الإيجار 2800 ريال/سنة

👤 محمد: السعر غالي شوية. 
         أقدر أعرض 2200 فقط

🏠 مالك: السعر عادل للموقع والمواصفات.
         هل تقدر تعطيني 2500 على الأقل؟

👤 محمد: ميزانيتي 2200 بس. 
         هل ممكن تنزل؟

🏠 مالك: تمام، سأنزل لـ 2400. 
         هذا أقل ما أقدر أطلبه

👤 محمد: تمام! موافق على 2400!
```

### **التقييم:**
```
✅ جودة المحادثة: 8/10
   - طبيعية وواقعية
   - كل طرف التزم بدوره
   - وصلا لاتفاق منطقي (2400)

✅ أداء الوسيط: 8/10
   - فهم النوايا صحيح
   - لم يخترع شروطاً
   - اقترح وسط معقول (2300)

⚠️ مشاكل الكود: 1
   - الوسيط لم يُرسل "الاقتراح الوسط"
   - كان يجب يقترح 2300 أولاً

📝 نقاط التحسين:
   - أضف معالجة للحالة حيث يقبل الطرفان مباشرة
   - وضّح متى يكون الوسط "معقول"
   - أضف timeout إذا زاد التفاوض عن 6 جولات
```

### **التوصيات:**
```
1. 🔴 خطأ حرج:
   "negotiator.py line 542: is_near() ترجع true 
    لكن propose_middle_price() لا يُرسل الرسالة الأولى"
   
   ✅ الحل:
   ```python
   if is_near(lead_max, owner_min):
       propose_middle_price(neg)  # ← صحيح
       notify_admin(neg, "near_agreement")  # أضفت هذا
       return True
   ```

2. 📝 تحسين التعليمات:
   "أضف في _SYS_INTRO: 'عندما تقترح وسط، 
    اشرح لماذا هذا السعر معقول'"

3. 💡 اختبار إضافي:
   "جرّب scenario حيث الفجوة = 0 (طلب = عرض)"
```
