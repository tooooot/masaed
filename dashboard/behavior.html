⚙️ السلوك البرمجي لمساعد (الكود الحتمي)

قرارات ومسارات مكتوبة في الكود — مضمونة ولا تتأثر بمزاج النموذج. المصدر: scraper/negotiator.py
🟣 الهوية/التعليمات ⚙️ السلوك البرمجي 🔬 المختبر
برمجي منطق ثابت بالكود — يحدّد متى ينقل/يقترح/يغلق وكيف يوجّه المحادثة. النموذج لا يقرّر هذه، فقط يصوغ الكلام داخلها.

1) آلة الحالات (FSM) — تفرض مسار التفاوض برمجي

الحالة = دالّة على بيانات الصف (الأسعار). الكود ينقل بين الحالات حتمياً؛ كل حالة لها هدف يُحقن في صياغة النموذج.
opening→ المستأجر/المالك يطرح سعراً →relay
relay→ الطرف الآخر يطرح سعراً →converge
converge→ الفجوة صغيرة (اقتراح وسط) →closing
closing→ موافقة الطرفين →agreed ✅
(في أي وقت: إلغاء →cancelled)
الحالةالمعنىهدف مساعد المحقون
openingلا أسعار بعداستخرج رقماً صريحاً الآن
relayطرف واحد عرضاطلب رقم الآخر / طمئن الأول
convergeالسعران معروفان، فجوةقرّب واقترح وسطاً
closingسعر وسط مقترحاحصل على "موافق" صريحة
agreedاتفقاإغلاق تلقائي + رسالة إتمام
لتعديله قل لي: «عدّل عتبة "الفجوة الصغيرة" من 12% إلى 8%»، أو «أضف حالة "مراجعة الإدارة" قبل الإغلاق».

2) تصنيف نية الرسالة برمجي intent_parser.py

كل رسالة تُصنَّف أولاً (regex سريع ثم نموذج للغامض) إلى:
price_offer (مع المبلغ) | accept | reject | question | cancel | other
+ المشاعر (إيجابي/سلبي/محايد) + هل الموقف حازم؟
هذا التصنيف يقرّر أي مسار يأخذه الكود أدناه.

3) ترتيب المعالجة (سلسلة القرار) برمجي

لكل رسالة واردة، يمرّ الكود بهذا الترتيب — أول شرط يتحقق يتولّى:
1.  إلغاء صريح ("إلغاء/cancel")     → إغلاق + إبلاغ الطرفين
2.  موافقة على السعر المقترح        → سجّل القبول؛ لو الطرفان وافقا → إغلاق تلقائي 🎉
3.  موافقة صريحة (بلا سعر مقترح)     → إشعار الإدارة
4.  عرض سعر (price_offer + مبلغ)     → احفظه → تقارب؟ اقترح وسطاً : انقل للآخر
5.  رفض حازم للصفقة                  → إشعار "طرف ينسحب"
6.  طلب رقم الطرف الآخر              → رد ثابت: اشرح الوساطة (لا أرقام)
7.  طلب صور/معاينة                   → رد ثابت: أنسّق وأرجع لك
8.  إحباط/سب موجّه للبوت             → رد ثابت: إعادة تفاعل نحو السعر
9.  كثرة الرسائل (≥8)                → إشعار الإدارة
10. غير ذلك                          → النموذج يصوغ (ضمن هدف الحالة)
الخطوات 1-9 حتمية بالكامل (لا تمر على النموذج). الخطوة 10 فقط هي التي يصوغها النموذج.

4) القواعد الحتمية (ردود ثابتة بالكلمات المفتاحية) برمجي

إذا احتوت الرسالة إحدى هذه الكلمات → رد ثابت مضمون (لا هلوسة):
المُحفِّز (كلمات)الرد الثابت
اعطني رقم، رقمه، رقم المالك…«أنا الوسيط، ما يحتاج تبادل أرقام، وش آخر سعر؟»
صور، معاينة، أشوف، زيارة…يرسل الصور المخزّنة فعلياً 📷 + سؤال السعر؛ ولو طُلب إضافي → relay للطرف الآخر
لا تكرر، غبي، انقلع، فاشل…«آسف إذا ضايقتك، خلّنا نركّز على السعر»
لتعديله قل لي: «أضف كلمة "واتساب" لقائمة طلب الرقم»، أو «غيّر رد طلب الصور إلى ___».

5) قرارات الأرقام (التقارب والوسط) برمجي

"قريبان" إذا:  عرض المستأجر ≥ طلب المالك
            أو  الفجوة ≤ 1500 ر
            أو  الفجوة ≤ 12% من السعر المرجعي
السعر الوسط = تقريب((عرض + طلب) ÷ 2) لأقرب 500 ر
الإغلاق التلقائي = عند ضغط الطرفين "موافق" على السعر الوسط
لتعديله قل لي: «اجعل الوسط ينحاز للمالك 60%»، أو «لا تغلق تلقائياً، انتظر موافقة الإدارة».

6) حُرّاس إضافية برمجي

  • حارس التكرار: لا يرسل نفس رد البوت السابق حرفياً (يستبدله بسؤال عن السعر).
  • تصنيف الرفض المُصحّح: الإحباط/السب لا يُحسب رفضاً للصفقة (لا تصعيد خاطئ).
  • مهلة 20 ثانية + إعادة محاولة على كل نداء للنموذج (يمنع التعليق).
  • تسجيل الانتقالات: كل انتقال حالة يُكتب في السجل: [NEG #id] FSM: x → y

✅ مبني حديثاً برمجي

  • إرسال الصور: طلب الصور → يرسل مساعد صور/فيديو العقار المخزّنة فعلياً (Green API sendFileByUrl) ثم يسأل عن السعر.
  • جلب المعلومات الناقصة (relay): طلب صور إضافية / معلومة غير متوفرة → يعيد مساعد صياغة الطلب (DeepSeek) ويرسله للطرف الآخر عبر واتساب، ويخبر السائل «طلبتها وأوافيك».
  • 🧠 جدول الحافظ الذكي: ملف مضغوط لكل عميل يُحقن في كل رد بدل السياق العام:
    🧠 ملف العميل — محمد: • الدور: مالك + باحث | تفاوضاته: 3 (ناجحة: 1) • تفضيلاته: جدة · 3 غرف · ~40 ألف • ملاحظات: …
    حقائق فقط من قاعدة البيانات (بلا استدعاء LLM = بلا تكلفة).
  • 💸 DeepSeek فقط: كل النماذج الحيّة تعمل على DeepSeek (Claude مُعطّل لتوفير التكلفة).

📝 كيف تعطيني برومبت للإصلاح (برمجي)

إذا المشكلة في متى/كيف يتصرّف (نقل، إغلاق، أرقام، مسار، كلمات مفتاحية) → برمجي. قل لي بصيغة:
"في السلوك البرمجي: عند [حالة/كلمة] افعل [إجراء]"
أمثلة:
• "عند الوصول لـ closing، أرسل تنبيهاً للإدارة قبل الإغلاق"
• "أضف كلمات [...] لكشف طلب المعاينة"
• "غيّر عتبة التقارب إلى 8%"
• "لا تغلق الصفقة تلقائياً أبداً"
قاعدة بسيطة للتمييز: «ماذا يقول وكيف ينبره» = 🟣 تعليمات | «متى يفعل ماذا ويقرّر» = ⚙️ برمجي.
